# Security Policy

## Supported Versions

The following versions of Vun-Mail are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Vun-Mail, please follow these steps:

1. **Do not** create a public GitHub issue for the vulnerability.
2. Send an email to the project maintainer at [vinzhoost@gmail.com](mailto:vinzhoost@gmail.com) with the subject line "Vun-Mail Security Vulnerability".
3. Include the following information in your report:
   - A detailed description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Any possible mitigations you've identified

## Response Expectations

When you report a vulnerability:

- You will receive an acknowledgement of your report within 48 hours.
- We will investigate the issue and provide an estimated timeline for a fix within one week.
- If the vulnerability is accepted:
  - We will work on a fix and release it as soon as possible
  - You will be credited for the discovery (if you wish) in the release notes
- If the vulnerability is declined:
  - We will provide a detailed explanation of our decision
  - We're open to further discussion if you believe we've made an error

## Security Considerations

As Vun-Mail is a Telegram bot that handles temporary email addresses, we take security seriously. Key considerations include:

- Protection of user data and privacy
- Secure handling of API keys and tokens
- Prevention of abuse and spam
- Proper validation of all user inputs

We encourage responsible disclosure and appreciate the efforts of security researchers who help keep our project secure.
